Core Capabilities

What makes this system production-grade?

Dual-Channel Capture

We don't mix audio. The interviewer (Microphone) and Candidate (System Loopback) are recorded on separate tracks, ensuring 100% speaker separation accuracy.

Real-Time Intelligence

Utilizes Deepgram's streaming API for millisecond-latency transcription. The user sees captions instantly while the system tags "Speaker 00" vs "Speaker 01".

Asynchronous Workers

Heavy lifting (Fact Checking, LLM Analysis) is offloaded to Celery/Redis. This ensures the live WebSocket connection never lags or drops frames.

Resilience & Cleanup

Includes "Force-Kill" logic for stuck connections and exponential backoff for network drops. The system self-heals if Wi-Fi flickers.

High-Level Architecture

The component map of the entire system.

Client Side

Flutter app captures the mic. A separate Python .exe helper captures system audio. Both send binary PCM data to the server.

API Gateway

FastAPI acts as a pass-through. It authenticates users via Firebase, checks credits, and forwards streams to Deepgram.

Worker Swarm

Once the call ends, a job is pushed to Redis. Celery workers wake up, call Anthropic (Claude) for insights, and save results to Firestore.

Data Flow Cycle

How data moves from microphone to database.

1. Ingestion

Audio streams initiate WebSocket handshake. Credits are deducted. Connections to Deepgram established.

2. Live Loop

Audio Up -> Text Down. Server buffers finalized sentences in memory for the final report.

3. Handoff

User clicks "Analyze". Server merges both transcripts into one conversation history and kills connections.

4. Analysis

Worker grabs the job. Runs Speaker Inference -> Fact Mining -> Scorecard Generation -> DB Save.

System Sequence

Phase Breakdown

OS SPECIFIC

Windows Production Checklist

Critical steps for deploying the desktop client to end users.

Code Signing

Purchase an EV Certificate to sign the Flutter `.exe` and the Python helper. This prevents the scary "Windows protected your PC" SmartScreen popup.

Unified Installer

Use Inno Setup or MSIX to bundle the Flutter app and the Python `/dist` folder into a single install wizard that sets paths correctly.

Auto-Updater

Integrate sparkle or msix-updater to push bug fixes. Critical for beta software where helper binaries might need frequent patching.

AV Allow-listing

Submit the compiled `speaker_helper.exe` to Microsoft Security Intelligence to prevent false positive flags from Windows Defender.
